# HTML Developer Workbench

## Architecture

Data flow: **Shell scripts → JSON files → HTML pages (fetch + render)**

```
/var/www/html/workbench/
├── index.html               # Landing page
├── jira-detail.html         # 4-pane JIRA detail
├── assets/palette.css       # Design tokens (from agentforge)
├── data/
│   ├── worktrees.json       # Generated by scripts/fetch-worktrees.sh
│   ├── sprint.json          # Generated by scripts/fetch-jira.sh
│   └── jira/<AV-ID>.json   # Generated by scripts/fetch-jira-detail.sh
└── scripts/
    ├── fetch-worktrees.sh
    ├── fetch-jira.sh
    ├── fetch-jira-detail.sh
    └── update-all.sh

~/.cursor/skills/workbench-updater/SKILL.md  # Agent skill
```

## Landing Page (`index.html`)

Two sections, using the palette from [`assets/palette.css`](agentforge/assets/palette.css):

- **Worktree List**: Rendered from `data/worktrees.json`. Shows worktree name, current branch, last commit summary.
- **Sprint Board**: Rendered from `data/sprint.json`. Three columns — **To Do** (neutral), **In Progress** (clay), **Done/Open** (olive). Clicking an In Progress or Done card navigates to `jira-detail.html?id=AV-XXXXX`.

## JIRA Detail Page (`jira-detail.html?id=AV-XXXXX`)

Four tabbed panes, loaded from `data/jira/<AV-ID>.json`:

- **Code Approaches** — parsed from the `## Approaches` (or equivalent) section of `.cursor/td-docs/<AV-ID>.md` in the assigned worktree; rendered as a 3-column comparison grid (html-document comparison-grid pattern)
- **Implementation Plan** — parsed from the `## Implementation Plan` section; rendered as milestone timeline + risk table
- **PR Annotations** — parsed from `## PR Annotations` section + PR diff summary from GitHub API; rendered using html-code-review PR writeup pattern
- **Status** — PR status (open/merged/closed from GitHub REST API), DSR runs (`dsr list` filtered by branch), DFR runs (`dsr dfr` results)

## Data Scripts

### `scripts/fetch-worktrees.sh`
```bash
# Scan ~/workspace/*/; for each directory, check git remote for
# git@github-vcf.devops.broadcom.net:ANS/avi-dev.git
# Output: data/worktrees.json
```
Fields per worktree: `name`, `path`, `branch`, `last_commit`, `has_td_docs` (bool), `jira_ids` (from branch names)

### `scripts/fetch-jira.sh`
```bash
jira sprint --output json > /tmp/sprint_raw.json
# Transform: group issues by statusCategory.key → todo/in_progress/done
# Output: data/sprint.json
```

### `scripts/fetch-jira-detail.sh <AV-ID>`
```bash
# 1. Find assigned worktree (branch name contains AV-ID or td-docs file exists)
# 2. Read .cursor/td-docs/<AV-ID>.md → parse sections by ## headings
# 3. GitHub PR: GET https://github-vcf.devops.broadcom.net/api/v3/repos/ANS/avi-dev/pulls
#    using token from ~/.config/gh/hosts.yml (fallback: extract from git log)
# 4. DSR status: dsr list --count 20 --duration 7, filter by branch name
# Output: data/jira/<AV-ID>.json
```

### `scripts/update-all.sh`
Runs `fetch-worktrees.sh`, `fetch-jira.sh`, then `fetch-jira-detail.sh` for each In Progress/Done ticket.

## Design

Follows all rules from [`agentforge/assets/palette.css`](agentforge/assets/palette.css):
- Ivory background, slate text, clay for active/focus, olive for done/success
- Cards with `border: 1px solid var(--gray-300)`, `border-radius: var(--r-md)`
- Monospace for branch names and commit hashes
- Status badges: clay=In Progress, olive=Done, neutral=To Do, info=PR Open, warning=DSR Running

## Skill: `~/.cursor/skills/workbench-updater/SKILL.md`

Instructs an agent to:
1. Run `update-all.sh` to refresh all data
2. Run `fetch-jira-detail.sh <AV-ID>` to refresh a specific ticket
3. Manually populate `data/jira/<AV-ID>.json` when scripts can't reach APIs
4. Parse td-docs markdown and populate the JSON sections

## Key Notes

- GitHub API (`/api/v3/repos/ANS/avi-dev`) returns 404 with the current PAT — scripts will attempt the call and gracefully fall back to git-log based PR number extraction (`git log --oneline --grep="Merge pull request"`)
- DSR CLI works: `dsr list` returns a JSON array with fields `Request_ID`, `Branch`, `Status`, `Request_Type`, `Started_at`. `dsr get <ID>` returns full detail with `Info`, `Build jobs`, `Platform jobs`. Scripts will filter by branch name to associate DSR/DFR runs with a JIRA ticket.
- All HTML pages use `fetch()` with a CORS-safe local JSON approach; Apache/nginx should be able to serve the static files at `/var/www/html/workbench/`
- No external CDN or fonts — fully self-contained per the agentforge design constraints
